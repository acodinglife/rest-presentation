REST (architecture)

Both JSON and XML

URI like http://server.com/v1/billings

HTTP verbs : GET - POST - PUT - DELETE

URI instead of functions GET /v1/users

How to use ?

SOAP REST
GETALLUSERS GET /v1/users
GETBYID(id) GET /v1/users/{id}
ADDUSER POST /v1/users
DELETEUSER DELETE /v1/users/{id}
UPDATEUSER PUT /v1/users/{id}

Response code

What happened ?

3 major families

OK (200 family)

Client / Server error (400 family)

Server error (500 family)

All of the verbs are available here

Examples

Trying to access this ressource (the site doesn't exist) : 

						GET http://mySuperApp.com/api/v1/users
Returns this response code : 

						404 NOT FOUND
Signifying that the URI doesn't gives anything at all
Trying to access this ressource (does exist) : 

						GET http://mySuperApp.com/api/v1/articles
Returns this response code : 

						200 OK
With this content : 

						[
						{title:"My first article",content:"Here's the content of my blog article"}
						{title:"My second article",content:"Here's the content of my blog article"}
						]
Signifying that the request has been handled successfully and gives an output
